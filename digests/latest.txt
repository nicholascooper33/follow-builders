Latent Space — Extreme Harness Engineering for Token Billionaires: 1M LOC, 1B toks/day, 0% human code, 0% human review — Ryan Lopopolo, OpenAI Frontier & Symphony

Takeaway: A three-person team at OpenAI shipped a million-line codebase in five months using zero human-written code and zero human code review — and the key wasn't just the models, it was treating the entire build system as a harness optimized for agents.

Ryan Lopopolo works on frontier product exploration at OpenAI, building enterprise agent products. He imposed one hard constraint on himself from the start: write no code himself. If the agent couldn't do it, he had to give it better building blocks rather than stepping in. The first month and a half was ten times slower than working alone. But by paying that upfront cost, the team eventually outpaced any single engineer — running over a billion tokens per day, shipping 1,500 pull requests, with most "review" happening post-merge.

The central engineering discipline Lopopolo calls harness engineering: structuring your codebase, tooling, and SDLC so that agents can work within it reliably. This meant keeping builds under one minute (to fit within the agent's attention span), migrating from Make to Bazel to Turbo to NX in a single week to hit that target, and investing heavily in observability so the agent could see what was happening in the app. Every time human review became the bottleneck, the team asked why and automated their way out of it.

One key insight on verification: don't let agents spend tokens on open-ended verification that doesn't improve the output. Build tight feedback loops instead — observable outputs, fast builds, deterministic checks.

On scalability: "The only fundamentally scarce thing is the synchronous human attention of my team. There's only so many hours in the day." Tokens are cheap and infinitely parallelizable. Human attention isn't. The architecture follows from that constraint.

This episode is a defining piece of thinking on what it means to build software in the agentic era — not just using AI as a coding assistant, but redesigning the entire software development lifecycle around what agents can and can't do.
